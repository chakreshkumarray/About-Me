Q6.  Intermediate Stream and Terminal Stream?
Ans. Intermediate Stream: Intermediate operations are lazy (they don’t execute anything)

        Intermediate operations (like filter(), map())
        ➜ Only build the pipeline
        ➜ They return another stream
        ➜ No processing happens yet

        Intermediate operations define WHAT to do
         */

        List<Integer> list = Arrays.asList(2, 5, 8, 12);

        list.stream()
                .filter(n -> {
                    System.out.println("Filtering: " + n);
                    return n > 5;
                });  // ❌ No terminal operation

        // (No output)

     Terminal Stream: Terminal operations trigger the actual execution of the stream pipeline.

        Terminal operation (like forEach(), collect())
        ➜ Starts the execution
        ➜ Produces final result
        ➜ After this, stream is consumed

        Terminal operations actually DO it.
         */

        List<Integer> list = Arrays.asList(2, 5, 8, 12);
        list.stream()
                .filter(n -> {
                    System.out.println("Filtering: " + n);
                    return n > 5;
                })
                .forEach(System.out::println);

        /* (Output)

        Filtering: 2
        Filtering: 5
        Filtering: 8
        8
        Filtering: 12
        12

Q8. Drop, Delete and Truncate in Database?
Ans. DELETE
     👉 Removes specific rows
     DELETE FROM employees WHERE id = 101;
     ✅ Key Points:
    Removes selected rows
    Uses WHERE condition
    Can rollback (if transaction is used)
    Triggers are fired
    Slower (row-by-row deletion)

    TRUNCATE
    👉 Removes all rows (fast)
    TRUNCATE TABLE employees;
    ✅ Key Points:
    Removes all rows
    ❌ No WHERE clause
    Cannot rollback (in most DBs)
    Does NOT fire triggers
    Faster than DELETE (deallocates memory)
    Resets AUTO_INCREMENT

    DROP
    👉 Deletes entire table
    DROP TABLE employees;
    ✅ Key Points:
    Removes table + data + structure
    ❌ Cannot rollback
    Frees all memory
    Table no longer exists

q9. Authentication and Authorization?
Ans. 🔐 1. Authentication (AuthN)
     👉 “Who are you?”

     Authentication is the process of verifying identity.

     ✅ Examples:
     Logging in with username & password
     OTP verification
     Biometric (fingerprint, face)

    🔹 Real-life example:
     Entering ATM PIN → proves you are the account holder

    🔑 2. Authorization (AuthZ)
    👉 “What can you access?”

    Authorization happens after authentication and determines permissions.

    ✅ Examples:
    Admin can delete users
    User can only view their profile
    Access control in APIs
   🔹 Real-life example:
    After ATM login → you can withdraw money but cannot access someone else’s account

Q16. What is trigeer in Database?
Ans. A trigger is a special type of stored program that automatically executes (fires) when a specific event occurs on a table.
     👉 You don’t call a trigger manually — the database runs it automatically.
     
    🔹 When does a Trigger Fire?
     Triggers are executed on events like:
       INSERT
       UPDATE
       DELETE

      🔹 Types of Triggers
      1. BEFORE Trigger
      👉 Executes before the event

      2. AFTER Trigger
      👉 Executes after the event

      CREATE TRIGGER before_insert_employee
      BEFORE INSERT ON employees
      FOR EACH ROW
      SET NEW.salary = NEW.salary + 1000;
